# Security Policy

## Supported Versions

This project is a personal wiki. We prioritize security fixes for the latest version of the platform.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |

## Reporting a Vulnerability

I take the security of this repository seriously. If you discover any security vulnerabilities, please do not report them via public issues. Instead, please follow these steps:

1. **Email the Maintainer**: Send a detailed description of the vulnerability to [contact@suryaraj.com](mailto:contact@suryaraj.com).
2. **Details to Include**:
   - Description of the vulnerability.
   - Steps to reproduce.
   - Potential impact.
3. **Response Timeline**: You can expect an initial response within 48-72 hours. I will provide updates throughout the resolution process.
4. **Public Disclosure**: Please allow a reasonable amount of time for the vulnerability to be addressed before making any information public.

## Security Scope
This policy covers:
- Vulnerabilities in the Docusaurus configuration or custom scripts.
- Exposure of sensitive information in the public repository history.
- Critical data inaccuracies that could lead to financial or personal harm.

Thank you for helping keep this digital garden safe!
